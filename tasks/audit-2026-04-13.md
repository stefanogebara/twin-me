# TwinMe Full Platform Audit — 2026-04-13

## Summary
Comprehensive audit covering backend, security, frontend E2E, UI/UX, and mobile. Tested in demo mode via Playwright. Servers at localhost:3004 (backend) and localhost:8086 (frontend).

---

## CRITICAL

**C1 — /goals route silently redirects to /dashboard in demo mode**
Any user navigating to `/goals` gets bounced to the dashboard with no explanation. Route is inaccessible. Should show demo goals state or explain why.

**C2 — Settings page fires 14 real API calls with no demo-mode guard**
`/settings` makes live calls to: `/api/autonomy/settings`, `/api/user-rules`, `/api/telegram/status`, `/api/whatsapp-link/status`, `/api/dashboard/context`, `/api/billing/subscription`, `/api/feature-flags` — all 401 in demo mode. Noisy console flood and partial UI renders. Needs `if (isDemoMode) return demoData` before firing API calls.

---

## HIGH

**H1 — trigger-all endpoint missing admin guard**
`GET /api/test-pattern-learning/trigger-all` only requires `authenticateUser`, not `requireProfessor`. Any authenticated user can trigger a full cross-user LLM pipeline run.
File: `api/routes/test-pattern-learning.js:69`

**H2 — test-evidence-pipeline unauthenticated outside NODE_ENV=production**
`GET /api/test-evidence-pipeline/:userId` blocks only in production. Vercel preview deploys may not set this.
File: `api/routes/test-evidence-pipeline.js:11`

**H3 — discovery/scan rate limit resets on cold start**
`POST /api/discovery/scan` is public with in-memory rate limit fallback that resets on cold start. Allows email enumeration at scale.
File: `api/routes/discovery.js:69`

**H4 — Literal 'fallback' HMAC secret in emailService**
`generateUnsubscribeToken` uses `|| 'fallback'` if both env vars missing. Allows forging unsubscribe tokens.
File: `api/services/emailService.js:99`

**H5 — /api/feature-flags is dev-only but Settings calls it in all envs**
Route registered inside `if (NODE_ENV === 'development')`. Production Settings page gets silent 404s — feature flag toggles broken for all prod users.
File: `api/server.js:587-590`

---

## MEDIUM

**M1 — Mobile: PWA install prompt + Feedback button overlap chat input**
On 375px, both elements stack at the bottom and cover the message input. Input becomes hard to tap.

**M2 — Auth page has screenshot timeout (heavy animation)**
`/auth` consistently times out on screenshot. Suggests heavy animation/video blocking paint. Could hurt FCP on slow connections.

**M3 — /api/health/deep leaks DB error messages publicly**
Unauthenticated deep health endpoint returns raw DB error strings.
File: `api/routes/health.js:23`

**M4 — JWT blacklist not shared across serverless instances**
In-memory fallback blacklist means logged-out tokens stay valid on other cold-started Vercel instances.
File: `api/middleware/auth.js:10-31`

**M5 — billingRoutes used before import in server.js**
`app.use('/api/billing', billingRoutes)` at line 296 before import at line 418. Fragile ordering.
File: `api/server.js:296 vs 418`

## LOW / UX

**L1 — Wiki shows as "Knowledge Graph" with vague empty state**
`/wiki` title is "Knowledge Graph", nav says "Knowledge". Empty state: "still being compiled" with no progress or timeline. User doesn't know how far they are.

**L2 — Dashboard greeting says "Explorer" not demo user name**
"Good evening, Explorer" on dashboard but Identity page shows "Good Evening, Alex". Inconsistency.

**L3 — Dashboard Soul Signature card is bare empty state in demo**
Above-the-fold card says "Connect a few platforms..." even in demo mode where Identity shows full soul data.

**L4 — Departments: raw API cost display may confuse users**
`$0.00 / $0.60 monthly` and per-department cost visible upfront. Showing raw API cost creates anxiety before value is established. Consider tucking behind settings.

**L5 — Chat right panel shows empty states in demo**
Calendar: "No events today". Emails: "No recent emails". Should show demo content to illustrate the feature.

**L6 — /api/auth/me returns 404 not 401**
No `/me` route — auth is at `/api/auth/verify`. Any client calling `/me` gets a confusing 404.

**L7 — Chat endpoint has 300s timeout (5 minutes)**
Per CLAUDE.md Vercel cost rules, maxDuration should be 60s. 300s = 5x the cost on hangs.
File: `api/server.js:244`

**L8 — sk-placeholder in llmGateway trips secret scanners**
`'sk-placeholder-will-fail-at-call-time'` prefix triggers false positives in CI.
File: `api/services/llmGateway.js:231`

**L9 — Owner email hardcoded in source**
`process.env.OWNER_EMAIL || 'stefanogebara@gmail.com'`
File: `api/routes/admin-beta.js:33`

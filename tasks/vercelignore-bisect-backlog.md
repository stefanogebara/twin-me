## Bisection protocol

For each candidate `X`:

1. Create branch: `git checkout -b bisect/vercelignore-X`
2. Edit `.vercelignore` — add ONE line (`X/`) above the warning comment block
3. Commit + push: `git push -u origin bisect/vercelignore-X`
4. Wait ~13-15 min for Vercel preview to build
5. Get preview URL: `vercel ls --yes | grep bisect`
6. Probe via Vercel MCP (preview URLs are SSO-gated so plain `curl` returns 401):
   ```
   mcp__30d6196a-3e3a-4aaf-8bfd-890b101a63d8__web_fetch_vercel_url
     url: https://twin-ai-learn-<id>-stefanogebaras-projects.vercel.app/api/health
   ```
7. If response is 200 with `{"status":"ok"}` → **safe to merge**
8. If response is 500 with `FUNCTION_INVOCATION_FAILED` → culprit found, delete branch, add to the "never exclude" list in `.vercelignore`

Takes ~20 min per candidate (13m build + 5m probe + 2m cleanup). Budget: ~4 hours for all 12 remaining dirs, though you'll likely find the answer in the first 2-3.

## Expected gain

Build time currently ~13-14 min. Excluding the safe dirs could drop it to ~10-11 min (rough estimate based on total upload size reduction). Not a huge win per build, but compounds if you ship often.

Alternative: extract more standalone Vercel functions (like `api/webhook-pluggy.js`). That shrinks the lambda's trace graph without touching the ignore list at all. Higher-leverage long-term.

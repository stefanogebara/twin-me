### 1. [SECURITY] Secrets committed and pushed to GitHub
- Multiple markdown files contain LIVE credentials pushed to origin:
  - `GITHUB_CLIENT_SECRET` (2 different values)
  - `JWT_SECRET` (2 different values)

  - `TOKEN_ENCRYPTION_KEY`
- Files: OAUTH_CREDENTIALS_SUMMARY.md, OAUTH_INTEGRATION_GUIDE.md, OAUTH_FIX_COMPLETE.md, etc.
- **Fix**: Rotate ALL exposed credentials immediately, scrub git history with BFG/filter-repo, force push

### 2. [SECURITY] Azure OpenAI API key in plaintext file
- `AZURE_CREDENTIALS.txt` contains live Azure OpenAI key in working directory

### 3. [SECURITY] Stack trace leak in production error responses

- **Fix**: Strip `stack` field from error responses when NODE_ENV !== 'development'

### 4. [CONFIG] Production VITE_APP_URL points to localhost
- All Vercel prod URLs redirect to http://localhost:8086
- No real user can use the prod URL
- **Fix**: Update Vercel env var VITE_APP_URL to https://twin-ai-learn.vercel.app, redeploy

### 5. [CONFIG] Google OAuth callback only configured for localhost
- "Continue with Google" redirects callback to localhost:8086

### 6. [A11Y] Settings toggle switches lack role="switch" and aria-checked

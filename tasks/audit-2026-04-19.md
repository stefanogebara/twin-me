## HIGH

- [ ] **H1. Chat SSE: twin responses never render in UI** — SSE stream completes with no `chunk` events before `done`. Frontend marks every message as `failed: {errorType:'timeout'}`. Response IS generated server-side (visible in /brain). Root cause: ~55s latency exceeds some upstream timeout. The `firstChunk` guard in TalkToTwin.tsx:339-481 never flips. Fix: investigate why chunk events don't arrive before stream closes locally.
- [ ] **H2. Markdown rendered as raw text in /brain** — Memory cards show `**bold**`, `**46% recovery**` as literal asterisks instead of rendered text. `displayContent` in MemoryFeed is not passing through a markdown renderer. Fix: strip or render markdown in memory content display.
- [ ] **H3. Mobile nav missing /brain and /wiki** — BottomNav MORE_NAV only has Settings. /brain (Memories) and /wiki (Knowledge) have no mobile entry point at all. Users on mobile cannot access their memory stream. Fix: add Brain/Wiki to MORE_NAV or promote one to primary tabs.
- [ ] **H4. More drawer stays open across navigation** — Opening the bottom nav "More" drawer then navigating keeps the drawer visible on the new page, overlapping content. Fix: close drawer on route change in BottomNav.
- [ ] **H5. Security: raw err.message in production responses** — `api/routes/imports.js:229` returns `err.message` without NODE_ENV guard. `api/routes/github-connect.js:86` falls back to `err.message` for unexpected errors. Fix: replace with safe fallback strings.
- [ ] **H6. auth/refresh returning 400** — Repeated 400 errors from /api/auth/refresh suggest token refresh is failing, which causes silent auth degradation in long sessions.

## Bug Findings

### FAIL (3) ‚Äî Wrong Endpoint Paths in Audit Script

The audit script used paths that differ from actual production routes:

| Audited Path | Actual Production Path | Root Cause |
|---|---|---|
| `GET /memories/stats` | `GET /twin/memory-stats` | Wrong prefix ‚Äî route is under `/twin/` |
| `GET /twin/chat/history` | `GET /chat/history?conversationId=<id>` | Wrong prefix ‚Äî route is under `/chat/` |
| `GET /twin/chat/intro` | `GET /chat/intro` | Wrong prefix ‚Äî route is under `/chat/` |
| `POST /twin/chat` | `POST /chat/message` | Deprecated endpoint ‚Äî correctly returns 501 |
| `GET /goals/stats` | `GET /goals/summary` | `/stats` matches `/:id` param route ‚Üí 400 |

These are NOT production bugs ‚Äî the routes work correctly at their actual paths. The audit script paths were wrong.

### WARN ‚Äî `/twin/insights` Latency (~15s)

`GET /twin/insights` is slow. First cold-start attempt returned 504. Second attempt with 20s curl timeout succeeded. This endpoint is at risk under load or on lower Vercel tiers (10s function timeout on Hobby plan). Recommend caching the response (e.g., 5-min Redis TTL similar to memory-health).

### INFO ‚Äî Legacy Expert Labels in Memory Stream

`/memory-health` expertBreakdown shows fragmented labels from pre-Phase 5:
- `social_analyst` (64), `music_psychologist` (57), `Unknown` (27), `productivity_analyst` (11)
alongside the canonical 5: motivation_analyst, lifestyle_analyst, social_dynamics, personality_psychologist, cultural_identity.

These legacy reflections persist in the DB. Not a blocking bug, but affects the `readiness.reflection` score calculation.
